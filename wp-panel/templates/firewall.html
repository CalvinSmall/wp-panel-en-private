Security Defense

Security DefenseRule Description

Detection Scope

Auto-ban rules monitor access.log and error.log of all sites; WordPress suspicious access analysis additionally reads wp-security.log for local statistics and evidence collection only.

Trigger Conditions (hitting any rule below counts as 1 violation)

  • POST /wp-login.php — WordPress login brute force
  • POST /xmlrpc.php returns HTTP 403 — still requested after XML-RPC disabled (includes //xmlrpc.php variant)
  • returns HTTP 429 — rate limit exceeded
  • Sensitive file scanreturns 404 — e.g. .env、.git、wp-config.php、.sql, archives, etc.
  • 404 flood detection — any pathreturns 404 reaching 30 hits/60 seconds classified as directory scanning (independent rule, fixed threshold)
  • WordPress anomalous path access — written to wp-security.log, for admin analysis, no auto-ban

Ban Threshold

First 4 rules default to 60 secondswithin 5 hitsviolations → auto-ban (thresholds, window, and ban duration can be customized in Security Settings).

404 flood rule is fixed at 30 hits/60 seconds, not configurable.

Progressive Tiered Ban

  • firsthitsviolations → Fail2ban Ban 10 minutes, auto-released on expiry
  • 24 hourswithinhitsviolations → upgraded to 24 hours ban, also added to Nftables persistent rules
  • cumulative 3 hits 24 hoursBan → auto-converted toPermanent Ban, enforced only via Nftables persistent rules

The ban list can distinguish Fail2ban temporary bans from Nftables persistent bans via the "Source" column.

Ban Mechanism

  • Fail2ban — manages site protection bans (jail: wppanel、wppanel-404) and SSH protection bans (jail: wppanel-sshd), auto-released on expiry
  • Nftables Persistent — managesPermanent Ban、Scan DefenseBan（Source: panel_scan) and panel auth bans (source: panel），independent of Fail2ban, auto-released on expiry
  • Nginx Blocklist — integrated with Fail2ban，Web protection jail Ban, synced to Nginx geo blocklist，banned IP directlyreturns 444 (close connection), ensuring effective blocking even behind Cloudflare proxy

All three channels work simultaneously: Nftables blocks direct requests to origin, Nginx blocks requests forwarded through Cloudflare, Fail2ban auto-releases on expiry.

Cloudflare Scenario Notes

  • Panel auto-deploys /etc/nginx/conf.d/wppanel-realip.conf, derives real visitor IP from Cloudflare's CF-Connecting-IP header
  • Whitelist auto-updates official Cloudflare IP ranges on refresh, ensuring only Cloudflare sources are trusted
  • No manual configuration needed; auto-enabled when Cloudflare proxy is detected

To disable Nginx-layer blocking: delete /etc/fail2ban/action.d/wppanel-nginx.conf, and from /etc/fail2ban/jail.d/wppanel.conf remove the wppanel-nginx action, then systemctl reload fail2ban && nginx -s reload。

Scan Defense

  • Targets panel port 8443 ; does not apply to site ports (80/443)
  • Non-browser requests (e.g., curl, scripts, scanners) hitting panel port without matching random entry → banned 30 days; common browser system probing paths and requests with BasicAuth headers do not directly trigger scan ban
  • Hitting random entry (/{suffix}) allows requests through normally

UpUpon trigger, written to panel ban list, source labeled as "Scan Defense"，Levelas temporary 30 days。

Whitelist Exemption

127.0.0.1, Cloudflare / Googlebot / Bingbot official IP ranges, and custom whitelist IPs will not be banned. Whitelist is managed in Security Settings.

WordPress security log path whitelist only excludes paths like ownership verification files; does not affect IP ban whitelist.

Rate Limiting Principle

Token bucket algorithm: one bucket per IP, capacity = burst buffer (default 300), second refills 1 token. Requests consume tokens; once depleted, returns 429. Human browsing has natural pauses, during which the bucket auto-refills; bots are limited to 1 hit/second.

⚠ multiplehits 429 will trigger a Fail2ban auto-ban.Please ensure the burst buffer is sufficient for normal usage.

✓ Increasing burst buffer does not affect bot blocking or weaken security.

Server Verification Commands

SSH into the server and run the following commands to check current ban status:

# Check site protection bans
fail2ban-client status wppanel
# Check 404 flood bans
fail2ban-client status wppanel-404
# Check SSH brute-force bans
fail2ban-client status wppanel-sshd
# View Nftables PersistentBan（Scan Defense + PanelAuthBan）
nft list set ip wppanel_persist banned_ips
