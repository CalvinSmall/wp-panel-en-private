Remote BackupUsage Guide

                        
                    

                    

                        

                            
Workflow

                            
    
                                
  1. Panel completes local database or file backup
    2. 
                                
  2. Automatically transfers backup files to remote location via SSH + rsync or S3-compatible API
    3. 
                                
  3. If "Keep Local Backup" is off, after successful transfer, delete local files to save space
    4. 
                            

                        

                        

                            
Remote Server Requirements

                            
    
                                
  • SSH service enabled and reachable from this server
    • 
                                
  • Target directory exists and backup account has write permission
    • 
                                
  • Firewall allows SSH port (default 22)
    • 
                            

                        

                        

                            
Auth Method

                            
    
                                
  • SSH Key (Recommended)：The panel auto-generates a keypair; the one-liner from the assistant on the right includes the public key. Just paste and run on the remote server. No extra software needed.
    • 
                                
  • Password：panel auto-installs sshpass when saving settings. n。
    • 
                                
  • S3-Compatible Object Storage：Uses Access Key and Secret Key for signed uploads; no remote SSH account needed. Large files use automatic multipart upload.
    • 
                            

                        

                        

                            
Security Recommendations

                            
    
                                
  • atRemote ServerCreatededicated backup accounton the remote server; do not use root. Least privilege is safer.
    • 
                                
  • Remote Backupdirectory does not auto-purge old files，periodic manual checks recommended
    • 
                                
  • Key file located at  /www/server/panel/remote_backup_key; keep it secure.
    • 
                            

                        

                        

                            
Tip: The one-liner generated by the assistant on the right can be copied and pasted directly into the remote server SSH terminal. Press Enter to complete all configuration.
